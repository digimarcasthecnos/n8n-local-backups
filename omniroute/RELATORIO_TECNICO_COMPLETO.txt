 11. AUTENTICACAO PROGRAMATICA - POST /api/auth/login (NOVA ETAPA)
--------------------------------------------------------------------------------
OBJETIVO DA ETAPA
  Verificar se o OmniRoute tem endpoint de login programatico (POST com senha) que
  substitua a extracao manual do cookie auth_token via DevTools. RESPOSTA: SIM, EXISTE.

METODO DE INVESTIGACAO (evidencia, nao suposicao)
  1. Inspecao das rotas compiladas dentro do container:
       docker exec omniroute sh -c 'find / -type d -path "*api*auth*"'
     -> Encontradas: /api/auth/login, /api/auth/logout, /api/auth/status
  2. Leitura do handler compilado (.next/server/.../auth/login/route.js + chunks):
       chunk src_lib_auth_managementPassword_ts -> funcoes verifyManagementPassword,
       hashManagementPassword, getStoredManagementPassword, ensurePersistent...Hash.
  3. Sondagem ao vivo do endpoint (sem precisar da senha real).

MODELO DE AUTENTICACAO DESCOBERTO
  - Senha de GESTAO UNICA (NAO ha usuario/email; so a senha).
  - Armazenada como HASH BCRYPT no data dir (regex $2[aby]$..); semente inicial via
    env INITIAL_PASSWORD no 1o setup (depois migra para hash persistido).
  - JWT_SECRET: NAO esta no env do container (so DATA_DIR, PORT, NODE_ENV, etc.) ->
    e gerado e persistido automaticamente pelo proprio OmniRoute no data dir.
  - Volume de dados: n8n-local_omniroute_data -> /root/.omniroute.

CONTRATO DO ENDPOINT
  Rota:    POST /api/auth/login
  Body:    {"password":"<senha-de-gestao>"}   (Content-Type: application/json)
  200:     seta cookie auth_token (JWT) no header Set-Cookie  -> e o cookie da sessao
  400:     campo password faltando         -> {"field":"password","expected string..."}
  400:     campo password vazio            -> {"message":"Password is required"}
  401:     senha errada                    -> {"error":"Invalid password"}
  429:     muitas tentativas               -> {"error":"Too many failed attempts..."}
  500:     JWT_SECRET ausente              -> nao ocorre no setup padrao

PROVA AO VIVO (executada nesta sessao)
  POST {}                              -> HTTP 400 (password: expected string, received undefined)
  POST {"password":""}                 -> HTTP 400 (Password is required)
  POST {"password":"senha-errada..."}  -> HTTP 401 ({"error":"Invalid password"})
  (Confirma que o endpoint existe, valida o campo "password" e responde como esperado.
   Com a senha correta retorna 200 + Set-Cookie auth_token.)

USO PROGRAMATICO (PowerShell) - captura automatica do cookie
  $b = @{ password = (Read-Host "Senha de gestao") } | ConvertTo-Json
  $r = Invoke-WebRequest -Uri "http://localhost:20129/api/auth/login" -Method Post `
         -ContentType "application/json" -Body $b -SessionVariable sv -TimeoutSec 15
  $tok = ($sv.Cookies.GetCookies("http://localhost:20129") |
          Where-Object Name -eq "auth_token").Value
  Set-Content -Path "omniroute_audit\.omni_cookie" -Value "auth_token=$tok" -Encoding ascii
  # Porta 20129 = instancia Docker (n8n). Na instancia nativa Windows (Parte 2) usar 20128.
