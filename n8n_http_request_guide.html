Opções de resposta
Tipo de autenticação

Nenhum cabeçalho de autenticação será enviado.

O n8n converte automaticamente para Authorization: Basic [base64]

Adiciona o header Authorization: Bearer <token> automaticamente.

Útil para APIs que usam headers como X-Api-Key ou x-access-token.

Dica: credentials do n8n

Em vez de digitar tokens diretamente, salve suas credenciais em Settings → Credentials e referencie-as no nó. Isso mantém as chaves seguras e fora do fluxo exportado.

Headers da requisição

Metadados enviados junto com a requisição. O mais comum: Content-Type: application/json

Parâmetros de query (URL params)

Adicionados ao final da URL: ?chave=valor&outra=coisa

Corpo da requisição

Disponível apenas em POST, PUT e PATCH.

Use {{ $json.campo }} para puxar valores do nó anterior no fluxo.

Nome do campo binário no item n8n (ex: data, attachment). O arquivo vem de um nó anterior (Read Binary File, HTTP Request, etc).

Dicas de erro comuns
401 Unauthorized — token errado ou expirado. Verifique a aba Auth.
403 Forbidden — sem permissão. O token não tem acesso ao recurso.
404 Not Found — URL incorreta. Verifique os parâmetros na URL.
422 Unprocessable — body inválido. Verifique os campos JSON enviados.
500 Server Error — erro no servidor da API. Teste a URL em outro cliente (ex: Postman).
SSL Error — ative "Ignorar erros SSL" na aba Básico (só para testes!).
